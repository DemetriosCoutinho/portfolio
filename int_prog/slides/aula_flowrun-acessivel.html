Capítulo 1 — Introdução: O que é programação?

O que é um programa de computador?

Imagine que você quer pedir uma pizza. Você liga para a pizzaria e dá uma série de instruções: "Quero uma pizza grande, sabor calabresa, para entrega no endereço tal." O computador funciona da mesma forma — ele executa instruções, uma por uma, na ordem que você definiu.

Definição: Programar é escrever instruções que o computador vai executar na ordem certa para resolver um problema.

O FlowRun e os fluxogramas

O FlowRun é uma ferramenta que permite criar programas usando fluxogramas — diagramas que mostram a sequência de ações de um programa. Cada bloco tem um formato diferente dependendo do que ele faz.

Os 5 símbolos do FlowRun

Oval (pílula)
Representa o Início e o Fim do programa. Todo fluxograma começa e termina com um oval.
Retângulo
Representa as operações Declare (criar uma variável) e Assign (atribuir um novo valor a uma variável).
Paralelogramo inclinado para a direita (Input)
Representa a entrada de dados. O programa pausa e espera o usuário digitar um valor.
Paralelogramo inclinado para a esquerda (Output)
Representa a saída de dados. O programa mostra um valor ou texto na tela.
Losango (diamante)
Representa decisões (If, While, For). Tem uma condição que pode ser verdadeira ou falsa, abrindo dois caminhos diferentes.

Quiz — Capítulo 1

Pergunta 1 — Qual símbolo representa uma DECISÃO no fluxograma do FlowRun?

O losango (diamante) representa decisões como if, while e for — tem uma condição que pode ser verdadeira ou falsa.

Pergunta 2 — Para mostrar um resultado na tela (Output), qual símbolo é usado?

O paralelogramo inclinado para a esquerda representa o bloco Output, que exibe valores na tela.

Pergunta 3 — O que é um programa de computador?

Um programa é uma lista de instruções ordenadas que o computador executa passo a passo para resolver um problema.

Capítulo 2 — Saída de Dados: Olá, Mundo!

O bloco Output

Definição: Output significa saída. É o bloco que mostra valores ou textos na tela do usuário.

Para usar o Output no FlowRun, coloque entre aspas duplas qualquer texto que queira exibir. Números não precisam de aspas.

Textos precisam de aspas duplas

Exemplos:

# Correto — texto com aspas duplas:
Output: "Olá, Mundo!"

# Correto — número sem aspas:
Output: 42
Output: 3.14

# Errado — texto sem aspas (causa erro):
Output: Olá, Mundo!

Fluxograma: Olá, Mundo!

Passos do fluxograma para o programa "Olá, Mundo!":

  1. Início (oval).
  2. Output: escrever na tela o texto "Olá, Mundo!" (paralelogramo de saída).
  3. Fim (oval).

Como criar este programa no FlowRun

  1. Acesse flowrun.io e crie um novo projeto.
  2. Clique na linha entre os blocos Begin e End para inserir um novo bloco.
  3. Selecione o bloco Output no menu que aparece.
  4. Clique no bloco Output e escreva "Olá, Mundo!" (com aspas duplas).
  5. Clique em Run para executar o programa e ver o resultado.

Quiz — Capítulo 2

Pergunta 1 — Qual bloco do FlowRun é usado para mostrar algo na tela?

O bloco Output exibe valores e textos na tela. É representado pelo paralelogramo inclinado para a esquerda.

Pergunta 2 — Como deve ser escrito um texto no bloco Output do FlowRun?

No FlowRun, textos (Strings) devem ser escritos entre aspas duplas. Sem aspas, o programa interpreta como um nome de variável.

Pergunta 3 — Qual é a ordem correta de execução do fluxograma "Olá, Mundo!"?

Todo programa começa com Begin (Início), executa as instruções no meio e termina com End (Fim). O Output fica entre eles.

Capítulo 3 — Variáveis: Guardando informações

O que é uma variável?

Pense em uma variável como uma caixinha com etiqueta. A etiqueta é o nome da variável e o conteúdo da caixa é o valor armazenado. Você pode ler o valor, substituí-lo por outro ou usá-lo em cálculos.

Tipos de dados

Tipos de variáveis no FlowRun
Tipo Descrição Exemplos
Integer Números inteiros (sem vírgula) 5, -23, 0, 100
Real Números com casas decimais 3.14, 0.5, -2.7
String Textos (entre aspas duplas) "Olá", "João", "abc"
Boolean Verdadeiro ou Falso true, false

Declare e Assign

Regra: Declare cria a variável e define seu tipo. Assign muda o valor depois. O valor inicial vai após o sinal de igual.

# Declare — cria a variável com valor inicial:
qtd: Integer = 3
preco: Real = 1.50

# Assign — altera o valor de uma variável existente:
qtd = 5

# Usando variável em expressão:
total: Real = qtd * preco

Fluxograma: cálculo de compra

Este fluxograma calcula o total de uma compra (quantidade × preço):

  1. Início (oval).
  2. Declare qtd: Integer = 3 — quantidade de itens (retângulo).
  3. Declare preco: Real = 1.50 — preço de cada item (retângulo).
  4. Declare total: Real = qtd * preco — calcula o total multiplicando quantidade por preço (retângulo).
  5. Output: mostrar na tela o valor de total (paralelogramo de saída). Resultado: 4.50.
  6. Fim (oval).

Cuidado com a divisão

A divisão entre dois valores inteiros (Integer) descarta o resto e retorna apenas a parte inteira. Para obter o resultado com decimais, use valores do tipo Real.

# Divisão inteira (descarta o resto):
5 / 2 = 2

# Divisão real (mantém os decimais):
5.0 / 2.0 = 2.5

Quiz — Capítulo 3

Pergunta 1 — Qual é o tipo correto para guardar a nota de um aluno como 8.5?

O tipo Real armazena números com casas decimais. Como 8.5 tem vírgula, o tipo correto é Real.

Pergunta 2 — No FlowRun, qual é o resultado de 7 / 2 com variáveis do tipo Integer?

A divisão entre inteiros descarta o resto. 7 ÷ 2 = 3 com resto 1. O resultado é 3, e o resto é perdido.

Pergunta 3 — Se temos qtd = 4 e preco = 2.50, qual é o valor de total = qtd * preco?

4 × 2.50 = 10.0. Multiplique a quantidade pelo preço: 4 grupos de 2.50 somam 10.0.

Capítulo 4 — Entrada de Dados: Conversando com o usuário

O bloco Input

Definição: Input significa entrada de dados. O bloco Input pausa o programa e espera o usuário digitar um valor, que é então guardado em uma variável.

Comparando Input e Output

Input (Entrada)
O usuário escreve algo que o programa captura e guarda em uma variável. O programa fica em pausa até o usuário confirmar a digitação.
Output (Saída)
O programa mostra um valor ou texto na tela para o usuário ler. O programa não aguarda resposta.

Fluxograma: lendo e mostrando um valor

Este fluxograma lê um número do usuário e o exibe de volta:

  1. Início (oval).
  2. Declare x: Integer — cria a variável sem valor inicial (retângulo).
  3. Input: ler o valor de x digitado pelo usuário (paralelogramo de entrada).
  4. Output: mostrar na tela "Você digitou: " + x (paralelogramo de saída).
  5. Fim (oval).

Concatenação: juntando textos com +

O operador + junta (concatena) textos e variáveis numa única frase de saída.

# Se x = 9:
Output: "Você digitou: " + x
# Resultado: Você digitou: 9

# Se nome = "Ana":
Output: "Olá, " + nome + "!"
# Resultado: Olá, Ana!

Quiz — Capítulo 4

Pergunta 1 — Se o usuário digita 9 em "Input: x", e depois executamos "Output: x * 2", o que aparece na tela?

O Input guarda o valor 9 em x. A expressão x * 2 calcula 9 × 2 = 18, que é o que o Output exibe.

Pergunta 2 — Qual é o resultado de "Olá, " + nome se o usuário digitou "Carlos"?

O operador + concatena o texto "Olá, " com o valor de nome (Carlos), produzindo Olá, Carlos. O resultado não tem aspas pois é a saída do programa.

Pergunta 3 — Para que serve o bloco Input no FlowRun?

O bloco Input pausa o programa e aguarda a digitação do usuário. O valor digitado é armazenado na variável indicada.

Capítulo 5 — Decisões: If e Else — caminhos diferentes

O bloco If

Pense assim: SE está chovendo, ENTÃO pego o guarda-chuva. O bloco If funciona da mesma forma — avalia uma condição e escolhe um caminho dependendo se ela é verdadeira ou falsa.

Fluxograma: positivo ou negativo

Este fluxograma lê um número e informa se ele é positivo ou negativo:

  1. Início (oval).
  2. Declare x: Integer (retângulo).
  3. Input: ler x do teclado (paralelogramo de entrada).
  4. Decisão (losango): a condição é x > 0?
    • Se sim (verdadeiro): Output "Positivo".
    • Se não (falso): Output "Negativo".
  5. Os dois caminhos se unem e o programa continua.
  6. Fim (oval).

Operadores de comparação

Operadores de comparação no FlowRun
Operador Significado Exemplo
>Maior quex > 0
<Menor quex < 0
>=Maior ou igual ax >= 5
<=Menor ou igual ax <= 10
==Igual ax == 7
!=Diferente dex != 0

Operadores lógicos

Operadores lógicos no FlowRun
Operador Nome Descrição Exemplo
&& E (AND) Verdadeiro só se AMBAS as condições forem verdadeiras x > 0 && y > 0
|| OU (OR) Verdadeiro se PELO MENOS UMA for verdadeira x > 0 || y > 0
! NÃO (NOT) Inverte o valor booleano (true vira false e vice-versa) !(x > 0)

Quiz — Capítulo 5

Pergunta 1 — Se o usuário digita x = -5, qual mensagem o fluxograma do If vai mostrar?

-5 não é maior que 0, então a condição x > 0 é falsa. O programa segue o caminho do "não" e exibe "Negativo".

Pergunta 2 — Para x = 8, qual é o resultado da condição: x > 5 && x < 10?

8 > 5 é verdadeiro E 8 < 10 também é verdadeiro. Com o operador && (E), ambas precisam ser verdadeiras — e são. Resultado: true.

Pergunta 3 — O que o operador ! (NOT) faz?

O operador ! (NOT) inverte o valor booleano. !(true) vira false, e !(false) vira true.

Capítulo 6 — Repetição: While e For — laços de repetição

Por que usar laços?

Conceito: Um laço executa um bloco de instruções várias vezes enquanto uma condição for verdadeira. Em vez de escrever o mesmo código 10 vezes, você escreve uma vez dentro do laço.

Fluxograma: While contando de 1 a 5

Este fluxograma usa o laço While para mostrar os números de 1 a 5:

  1. Início (oval).
  2. Declare x: Integer = 1 — inicializa o contador (retângulo).
  3. Decisão (losango): a condição é x <= 5?
    • Se não: ir para o Fim.
    • Se sim:
      1. Output: mostrar o valor de x.
      2. Assign: x = x + 1 — incrementa o contador.
      3. Voltar ao passo 3 (re-avaliar a condição).
  4. Fim (oval).

Saída do programa: 1, 2, 3, 4, 5.

Traço de execução do While

x = 1  → condição: 1 <= 5? sim → mostra 1 → x = 2
x = 2  → condição: 2 <= 5? sim → mostra 2 → x = 3
x = 3  → condição: 3 <= 5? sim → mostra 3 → x = 4
x = 4  → condição: 4 <= 5? sim → mostra 4 → x = 5
x = 5  → condição: 5 <= 5? sim → mostra 5 → x = 6
x = 6  → condição: 6 <= 5? não → para!

O laço For

O laço For tem um contador embutido — você define o valor inicial, o valor final e o passo (incremento). Ele é mais compacto quando o número de repetições é conhecido.

Fluxograma For para mostrar os números de 1 a 5:

  1. Início (oval).
  2. Laço For: i começa em 1, vai até 5, incrementa 1 a cada passo (losango especial de For).
    • Enquanto houver próximo valor de i: Output mostrar i; depois voltar ao início do For.
    • Quando i ultrapassar 5: ir para o Fim.
  3. Fim (oval).

Quando usar While e quando usar For

While
Use quando não sabe exatamente quantas repetições vão ocorrer. Exemplo: repetir até o usuário digitar um número válido.
For
Use quando sabe exatamente o número de repetições. Exemplo: percorrer 10 itens de uma lista ou imprimir uma tabuada.

Alerta: loop infinito

Cuidado: Se você esquecer de atualizar a variável do contador (como o x = x + 1), a condição nunca se tornará falsa e o programa vai repetir para sempre, travando. Sempre verifique se seu laço tem uma condição de saída!

Quiz — Capítulo 6

Pergunta 1 — No fluxograma While com x = 1 e condição x <= 5, quantas vezes o Output é executado?

O Output executa para x = 1, 2, 3, 4 e 5 — cinco vezes. Quando x chega a 6, a condição 6 <= 5 é falsa e o laço para.

Pergunta 2 — Qual a PRINCIPAL diferença entre While e For?

No For, o contador (início, fim e incremento) é configurado num único lugar. No While, você mesmo precisa declarar, incrementar e verificar a variável contadora.

Pergunta 3 — O que acontece se a condição do While NUNCA se tornar falsa?

Se a condição nunca vira falsa, o laço repete indefinidamente — o programa trava. Isso é chamado de loop infinito e é um erro comum de iniciantes.

Capítulo 7 — Pratique! Exercícios no FlowRun

Acesse o FlowRun

Abrir FlowRun para praticar →

Dica de ouro: Quando travar num exercício: desenhe o fluxograma no papel antes de montar no FlowRun! Pense nos passos um a um: o que entra? O que decide? O que sai?

Exercícios — Nível Básico

  1. Olá, Fulano! — Peça o nome do usuário com Input e cumprimente-o: "Olá, [nome]!"
    Blocos: Output, Input, String
  2. Dobro de um número — Peça um número inteiro e mostre o dobro.
    Blocos: Input, Output, Integer
  3. Positivo ou Negativo — Se o número for maior que zero, mostre "Positivo". Se não, mostre "Negativo ou Zero".
    Blocos: If, Input, Output
  4. Contador regressivo — Use um While para contar de 10 até 1 e mostrar "Fim!" ao terminar.
    Blocos: While, Assign, Output

Exercícios — Nível Intermediário

  1. Apenas números positivos — Use Do While para repetir a leitura até o usuário digitar um número positivo.
    Blocos: Do While, Input
  2. Tabuada — Peça um número N e imprima N×1, N×2, …, N×10.
    Blocos: For, Input, Output
  3. Soma de N números — Peça N, depois leia N números e mostre a soma total.
    Blocos: For, Input, Assign
  4. Maior e menor — Peça 5 números e descubra o maior e o menor entre eles.
    Blocos: While, If, Assign

Quiz — Capítulo 7

Pergunta 1 — Para o exercício "Tabuada", qual bloco de repetição é mais adequado?

A tabuada tem exatamente 10 multiplicações (N×1 até N×10). O For é ideal quando o número de repetições é conhecido de antemão.

Pergunta 2 — Para "Apenas números positivos" (repetir a leitura até digitar positivo), qual laço é mais adequado?

O Do While executa o corpo do laço pelo menos uma vez antes de verificar a condição — perfeito para "peça até ser válido", pois sempre precisamos ler pelo menos uma vez.

Pergunta 3 — Ao construir um fluxograma, qual é uma boa estratégia para começar?

Desenhar no papel antes ajuda a organizar o raciocínio: o que entra, o que decide e o que sai. Só depois leve para o FlowRun.
