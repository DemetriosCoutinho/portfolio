Objetivos da aula

  • Conhecer o conceito de algoritmos e suas formas de representação.
  • Aprender a resolver problemas de forma estruturada e lógica.

O que é um algoritmo

De forma informal, um algoritmo é um conjunto de passos — comandos ou instruções — para resolver um problema. Esses passos devem ser claros, precisos e sem ambiguidade: cada instrução deve ter exatamente um significado.

Definição formal: um algoritmo é uma sequência finita de instruções que resolve um problema específico.

O que é um problema

Na programação, um "problema" é qualquer situação que exige uma solução: um cálculo matemático, uma dúvida que precisamos responder, ou algo que é difícil de explicar sem um passo a passo.

Para cada problema pode haver várias soluções possíveis. O desafio é encontrar uma que seja correta, clara e eficiente.

Exemplo: vestir-se

Considere o problema de se vestir pela manhã. Um algoritmo possível é:

  1. Colocar a calça.
  2. Colocar as meias.
  3. Colocar os sapatos.
  4. Colocar a camisa.

Observação: a ordem importa. Inverter os passos 1 e 4 (colocar a camisa antes da calça) seria possível, mas inverter meias e sapatos não faria sentido. Isso ilustra que um algoritmo deve respeitar dependências entre as instruções.

Partes de um algoritmo

Todo algoritmo é composto de três partes fundamentais:

Entrada
Dados fornecidos ao algoritmo para que ele trabalhe. Exemplo: os dois números cuja média queremos calcular.
Processamento
As operações realizadas sobre os dados de entrada para produzir um resultado. Exemplo: somar os dois números e dividir por 2.
Saída
O resultado produzido pelo processamento, apresentado ao usuário. Exemplo: exibir o valor da média calculada.

Cinco características de um algoritmo

Para ser considerado um algoritmo válido, uma sequência de instruções deve ter cinco características:

  1. Finitude — o algoritmo deve terminar em algum momento. Não pode ficar em execução para sempre.
  2. Definição — cada passo deve ser precisamente definido, sem ambiguidade. O computador não interpreta — ele executa exatamente o que está escrito.
  3. Entradas — o algoritmo pode ter zero ou mais valores de entrada, fornecidos pelo usuário ou por outro programa.
  4. Saídas — o algoritmo deve produzir ao menos um resultado — a resposta ao problema.
  5. Efetividade — cada instrução deve ser suficientemente simples para poder ser executada, em princípio, de forma exata e em tempo finito.

Formas de representação

Um algoritmo pode ser descrito de quatro formas principais:

  1. Linguagem natural — escrito em português (ou qualquer idioma) como um texto comum.
  2. Fluxograma — diagrama com símbolos padronizados que representam cada passo visualmente.
  3. Pseudo-linguagem (Portugol) — texto estruturado que imita uma linguagem de programação, usando palavras em português.
  4. Linguagem de programação — código que pode ser executado por computador diretamente, como Java, C, C++, JavaScript, Python ou C#.

Linguagem de máquina

O computador só entende linguagem de máquina: sequências de zeros e uns (sistema binário). Cada grupo de 8 dígitos binários forma um byte e representa um valor — número, letra ou instrução.

Exemplos de representação binária
Binário Valor
000000000
000000011
000000102
01000011'C'
01000101'E'

Linguagens de programação

Como escrever código em binário seria impossível na prática, foram criadas as linguagens de programação: conjuntos de termos e regras que os humanos conseguem ler e escrever.

O código-fonte escrito pelo programador passa por um programa chamado compilador (ou interpretador), que o traduz para linguagem de máquina — binário que o computador pode executar.

Exemplos de linguagens de programação: Java, C, C++, JavaScript, Python, C#.

Linguagem natural

A linguagem natural é a forma mais simples de representar um algoritmo: escrevemos os passos em português corrente, como uma receita de bolo ou um manual de instruções.

Vantagem

Todo falante do idioma entende. Não é necessário aprender nenhuma notação especial.

Desvantagem

A linguagem natural é ambígua e imprecisa. "Leve ao forno até assar" tem um significado diferente para cada pessoa. Um computador não consegue interpretar esse tipo de instrução sem um significado único e exato.

Exemplo: treino de academia

Um algoritmo em linguagem natural para um treino poderia ser: "Aqueça por 10 minutos. Faça 3 séries de 15 repetições de supino. Descanse 60 segundos entre as séries. Alongue ao final." Claro o suficiente para um humano, mas impreciso demais para um computador.

Exemplo: lobo, bode e alfafa

Um fazendeiro precisa atravessar um rio com um lobo, um bode e um fardo de alfafa usando um barco que comporta apenas ele e mais um item. O lobo come o bode se ficar sozinho com ele; o bode come a alfafa se ficar sozinho com ela. Como o fazendeiro atravessa tudo sem perdas?

Solução (em linguagem natural)

  1. Leve o bode para o outro lado.
  2. Volte sozinho.
  3. Leve o lobo para o outro lado.
  4. Volte com o bode.
  5. Leve a alfafa para o outro lado.
  6. Volte sozinho.
  7. Leve o bode para o outro lado.

Fluxograma — definição

Um fluxograma é uma representação gráfica de um algoritmo usando figuras padronizadas internacionalmente. Cada símbolo representa um tipo de operação e setas indicam a direção do fluxo de execução.

Vantagens

Usa figuras em vez de texto puro, o que facilita a visualização do fluxo. Os símbolos são padronizados — qualquer pessoa treinada em qualquer país lê o mesmo fluxograma.

Desvantagens

Poucos recursos para descrever os tipos de dados envolvidos. À medida que o algoritmo cresce, o fluxograma fica grande e difícil de acompanhar.

Símbolos do fluxograma

Os seis símbolos fundamentais e seus significados:

Terminal (oval)
Marca o Início e o Fim do algoritmo. Todo fluxograma começa e termina com este símbolo.
Processo (retângulo)
Representa uma operação ou atribuição: cálculos, atribuição de valores a variáveis, qualquer ação que transforma dados.
Decisão (losango)
Representa uma condição com duas saídas: SIM (verdadeiro) e NÃO (falso). O fluxo se bifurca e segue por caminhos diferentes dependendo da resposta.
Entrada / Saída (paralelogramo)
Representa a leitura de um dado fornecido pelo usuário (entrada) ou a exibição de um resultado na tela (saída).
Conector (círculo)
Usado para juntar ou continuar fluxos em fluxogramas grandes, evitando cruzamento de setas.
Seta (linha com ponta)
Indica a direção do fluxo de execução — de onde vem e para onde vai o próximo passo.

Problema: média do aluno

Dado dois valores de notas de um aluno (N1 e N2), calcular e exibir a média aritmética entre elas. A fórmula é:

Média = (N1 + N2) / 2

Fluxograma da média (simples)

O fluxo do algoritmo para calcular a média simples, descrito passo a passo:

  1. Início — o programa começa.
  2. Entrada — ler os valores de N1 e N2 fornecidos pelo usuário.
  3. Processo — calcular: Média recebe (N1 + N2) / 2.
  4. Saída — exibir o valor de Média na tela.
  5. Fim — o programa encerra.

Fluxograma da média com aprovação

Esta versão adiciona uma decisão após o cálculo da média: verificar se o aluno foi aprovado (média maior ou igual a 6). O fluxo completo:

  1. Início — o programa começa.
  2. Entrada — ler N1 e N2.
  3. Processo — calcular: Média recebe (N1 + N2) / 2.
  4. Saída — exibir Média.
  5. Decisão — "Média ≥ 6?"
    • SIM → Saída: exibir "Aprovado".
    • NÃO → Saída: exibir "Reprovado".
  6. Fim — o programa encerra.

Problema das 9 bolas

Você tem 9 bolas aparentemente idênticas, mas uma delas é mais leve que as outras. Usando uma balança de dois pratos (sem pesos), descubra qual é a bola mais leve com o menor número possível de pesagens.

Solução

  1. Divida as 9 bolas em três grupos de 3: grupo A, grupo B e grupo C.
  2. 1ª pesagem: coloque o grupo A num prato e o grupo B no outro.
    • Se um dos pratos subir (ficar mais alto), o grupo mais leve está nele — a bola especial está nesse grupo.
    • Se os pratos ficarem equilibrados, a bola especial está no grupo C.
  3. 2ª pesagem: dentro do grupo identificado como o mais leve, pese duas das três bolas (uma de cada lado).
    • Se um prato subir, a bola naquele prato é a mais leve.
    • Se ficarem equilibradas, a bola não pesada é a mais leve.

Resultado: apenas 2 pesagens são suficientes para encontrar a bola mais leve entre 9. Este é um exemplo de raciocínio algorítmico eficiente — dividir o problema em partes menores a cada passo.

Pseudocódigo (Portugol)

O pseudocódigo é uma forma de representar algoritmos usando uma linguagem parecida com as linguagens de programação reais, mas com palavras em português. O Portugol é o pseudocódigo mais usado no ensino de programação no Brasil.

Vantagens

  • Mais próximo de uma linguagem de programação real do que a linguagem natural.
  • Permite descrever tipos de dados e estruturas de controle com precisão.
  • Facilita a transição para uma linguagem de programação de verdade.

Desvantagem

É um pseudocódigo — não pode ser executado diretamente por um computador. Não existe um padrão único: diferentes livros e professores usam notações ligeiramente diferentes.

Código em Portugol — cálculo da média

O algoritmo da média simples escrito em Portugol:

algoritmo "CalcularMedia1"
var
  N1, N2, Media : real
Inicio
  leia(N1, N2)
  Media := (N1 + N2) / 2
  escreva(Media)
fimalgoritmo

Leitura do código: declara três variáveis do tipo real (N1, N2 e Media). Lê N1 e N2 do teclado. Calcula a média e armazena em Media. Exibe o valor de Media na tela.

Exercícios propostos

Escreva um algoritmo (em qualquer representação vista em aula) para cada problema abaixo:

  1. Calcular a área de um terreno retangular a partir do comprimento e da largura informados pelo usuário.
  2. Calcular a idade atual de uma pessoa a partir do ano de nascimento informado pelo usuário.
  3. Exibir a tabuada completa (de 1 a 10) de um número informado pelo usuário.

Exercício: Torre de Hanói

A Torre de Hanói é um problema clássico que exercita raciocínio recursivo. Há três pinos (A, B e C) e três discos de tamanhos diferentes empilhados no pino A, do maior (embaixo) ao menor (em cima). O objetivo é mover todos os discos do pino A para o pino B ou C, obedecendo às três regras abaixo:

  1. Você pode mover apenas um disco por vez.
  2. Só é possível mover o disco que está no topo de uma pilha.
  3. Um disco maior nunca pode ser colocado sobre um disco menor.

Desafio: tente resolver para 3 discos e anote os passos como um algoritmo. Qual é o número mínimo de movimentos necessários?
