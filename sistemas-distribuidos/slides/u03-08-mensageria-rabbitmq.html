Bloco 2 · Desacoplamento

Três consequências do desacoplamento.

— 01 · Resiliência

Consumer cai, fila preserva

O producer não sente a queda. A mensagem fica na fila até o consumer voltar.

— 02 · Escalabilidade

Adicione workers sem tocar o producer

Novos consumers se conectam à mesma fila. O producer não precisa saber que existem.

— 03 · Independência

Serviços evoluem no próprio ritmo

Estoque e Notificação podem ser deployados, atualizados e reiniciados de forma separada.

▲ Trade-off: consistência eventual — o producer não sabe se o consumer processou. Se o cliente precisa do resultado imediatamente, HTTP ainda é a escolha certa.

Bloco 5 · Producer

gateway.py — quem publica o pedido.

▸ gateway.py 
from flask import Flask, jsonify, request   # framework HTTP
import pika, json

app = Flask(__name__)

def get_channel():
    conn = pika.BlockingConnection(              # producer: conexão TCP
        pika.ConnectionParameters('localhost'))
    ch = conn.channel()
    ch.exchange_declare(                         # cria a exchange
        exchange='pedidos',
        exchange_type='direct')
    return conn, ch

@app.route('/pedido', methods=['POST'])
def criar_pedido():
    pedido = request.get_json()
    conn, ch = get_channel()
    body = json.dumps(pedido)
    ch.basic_publish(exchange='pedidos',        # publica → estoque
                     routing_key='estoque',
                     body=body)
    ch.basic_publish(exchange='pedidos',        # publica → notificacao
                     routing_key='notificacao',
                     body=body)
    conn.close()
    return jsonify({'status': 'ok'}), 201      # retorna sem esperar

Bloco 5 · Consumer

Consumer: declarar fila, binding, callback.

▸ estoque.py 
import pika, json

def callback(ch, method, props, body):
    p = json.loads(body)
    print(f"📦 Estoque: {p}")

conn = pika.BlockingConnection(
    pika.ConnectionParameters('localhost'))
ch = conn.channel()
ch.exchange_declare(
    exchange='pedidos',
    exchange_type='direct')
# declara e faz binding da fila
ch.queue_declare(queue='estoque')
ch.queue_bind(exchange='pedidos',
             queue='estoque',
             routing_key='estoque')
ch.basic_consume(
    queue='estoque',
    on_message_callback=callback,
    auto_ack=True)
print('Aguardando...')
ch.start_consuming()

Bloco 7 · Síntese

O que você construiu hoje.

— Producer

Publica sem esperar

basic_publish + routing_key — o gateway retorna 201 antes dos consumers.

— Exchange

Roteia por routing key

Tipo direct: cada routing_key entrega apenas na fila correspondente.

— Consumer

Processa independente

queue_declare + queue_bind + basic_consume — o padrão é replicável.
